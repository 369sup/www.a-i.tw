# 產品概覽

> 此模型已核准為 in-memory 垂直切片。Production identity provider、database 與
> durable messaging 仍不在目前範圍。

產品探索的核心是將操作主體、資源 owner 與協作工作空間分離：

1. Principal 回答「誰提出請求」。
2. Account 回答「資源以誰的名義擁有」。
3. Repository 回答「哪個工作空間承載協作與政策」。

這個邊界讓登入、組織成員關係與資源範圍存取能獨立演進。

目前可在 [產品工作區](/workspace) 操作 Account 與 Repository governance。
