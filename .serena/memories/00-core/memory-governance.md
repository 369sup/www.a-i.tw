# Memory Governance

## Purpose

Define how Serena memories remain concise, source-backed, and maintainable.

## Summary

Memories are version-controlled navigation artifacts. They point to formal docs and current code evidence; they are not a second documentation system.

## Rules

- Use the required memory structure and `mem:` links.
- Keep stable, non-obvious facts only; place temporary investigation in `_temporary/`.
- Update status memories only after diagnostics, tests, and diff evidence are available.
