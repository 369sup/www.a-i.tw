# GitHub official non-code research routing

Canonical source routing: `docs/product/github-official-research-map.md`.
Closest instructions: `docs/product/AGENTS.md`.

For GitHub product research, select one official GitHub Docs section and one local canonical semantic owner. Extract official term, behavior, limitation and URL; classify it as Entity, Value, Relationship, Policy, Projection or Use Case. Official docs are external behavior evidence, not proof of local runtime.

Exclude Git, Commit, Branch, Tag, Merge, Pull Request, Diff, Code Review, source-code browsing/editing/search/navigation, Code Search, Actions/workflows/runners, Packages, Codespaces, Git LFS and code-centric security scanning.

Keep Enterprise, Entitlement, Notification and Search in separate owner documents. Never infer Domain hierarchy from GitHub Docs navigation.
