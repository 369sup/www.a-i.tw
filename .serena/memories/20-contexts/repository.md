# Repository Context

## Purpose

Route analysis of repository visibility, grant, lifecycle, and repository-scoped decision language.

## Summary

Repository is a proposed strategic context for product resource-container governance. It excludes Git, source, commit, branch, PR, Actions, package, and code-security semantics.

## Rules

- Do not conflate a product Repository with a filesystem folder or Git adapter.
- Require an explicit contract before another context consumes repository policy.
