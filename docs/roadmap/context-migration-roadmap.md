# Context migration roadmap

狀態：Proposed

1. Approve product owner, actor/problem and first vertical slice.
2. Approve one Context's language, Aggregate, use case, ports, data and contract through ADR.
3. Scaffold one module and update the runtime Context Map.
4. Add a second Context only with published language, ACL and consistency behavior.
5. Consider service extraction only with independent deployment, scaling, availability, compliance or ownership evidence.
