# Ubiquitous language

狀態：Baseline / Context-owned

| 中文名稱             | Code Name                           | 定義                                                                                                            | Owner Context          | 允許同義詞                     | 禁止詞                             |
| -------------------- | ----------------------------------- | --------------------------------------------------------------------------------------------------------------- | ---------------------- | ------------------------------ | ---------------------------------- |
| 主體                 | Principal                           | 可被驗證並可提出操作請求、可供 audit attribution 的 actor。                                                     | Identity & Access      | actor                          | user（泛稱）、account              |
| 驗證身分             | Authentication Identity             | Principal 與一種驗證方法或外部 provider subject 的 linkage。                                                    | Identity & Access      | login identity                 | account、credential                |
| credential           | Credential                          | 證明或代理 Principal 的 authentication mechanism；不是 identity、Account 或 resource grant。                    | Identity & Access      | authentication credential      | token permission、account          |
| 驗證情境             | Authentication Context              | 請求中可安全使用的已驗證 Principal、assurance 與 credential validity facts。                                    | Identity & Access      | authenticated principal        | repository permission              |
| 登入                 | Login                               | 使用 credential 證明 Principal 並請求建立 Session 的 use case；不是 Profile 或 authorization。                  | Identity & Access      | sign in                        | account switch、profile            |
| 工作階段             | Session                             | 以 opaque token 延續已驗證 Principal 的可撤銷瀏覽器狀態。                                                       | Identity & Access      | authenticated session          | cookie value、account              |
| 工作階段識別         | Session Identity                    | Identity owner 內部的穩定 Session identity；與 browser cookie 中的 opaque lookup token 分離。                   | Identity & Access      | session id                     | cookie token、credential           |
| 工作階段狀態         | Session Status                      | Session 的 `active`／`revoked`／`expired` lifecycle；終止狀態不得恢復 active。                                  | Identity & Access      | session lifecycle              | principal status、cookie state     |
| 工作階段期限         | Session Expiry                      | Session 可被解析為 authentication context 的最晚時間；current browser policy 為 authentication 後八小時。       | Identity & Access      | session deadline               | cookie max-age、credential expiry  |
| 驗證方法             | Authentication Method               | 本次 authentication 使用的 credential mechanism；current slice 為 `password`，不是 assurance 或 authorization。 | Identity & Access      | sign-in method                 | role、permission、assurance        |
| 個人資料             | Profile                             | Account 的可見表示；不含 credential、Session 或 resource grant。                                                | Account                | account profile                | principal、dashboard               |
| 個人資料顯示名稱     | Profile Display Name                | Profile 的必要公開展示名稱；可變且不改變 Account identity、handle 或 authorization。                            | Account                | display name                   | username、account handle           |
| 個人資料簡介         | Profile Bio                         | Profile 的可選公開文字，canonical trim 後最多 160 characters。                                                  | Account                | bio                            | description metadata、status       |
| 個人資料網站         | Profile Website                     | Profile 的可選 absolute HTTP(S) URL；HTTP(S) 限制是本產品安全 invariant，不冒充 GitHub 公開長度規則。           | Account                | website URL                    | callback URL、credential URL       |
| 個人面板             | Personal Dashboard                  | 登入後組合目前 Principal/Account、關係、存取與個人化 read models 的操作入口。                                   | Experience             | authenticated home             | profile                            |
| 活動行為者           | Active Actor                        | 目前 Session 用來歸屬行為的 User account；只能是 Personal 或 Managed user account。                             | Identity & Access      | actor context                  | organization account               |
| 活動範圍             | Active Scope                        | 目前 UI/Use Case 所在的 Personal、Organization 或 Enterprise governance scope；不取代 Actor。                   | Account                | selected scope                 | active account、workspace context  |
| 行為者切換           | Actor Context Switching             | 在已驗證 User sessions 間切換 Active Actor，需重新解析關係、政策與個人面板。                                    | Identity & Access      | account switcher               | organization switch                |
| 範圍切換             | Scope Context Switching             | Actor 不變時切換 Account 或 Governance Scope，並重新計算可見資料與功能。                                        | Account                | scope switcher                 | login、workspace switcher          |
| 請求封套             | Request Envelope                    | 所有 capability resolver 可使用的最小 viewer、actor、safe credential metadata、correlation 與 scope reference。 | Experience             | request facts                  | universal context service          |
| 能力上下文           | Capability Context                  | 由特定 capability family 擁有、在 envelope 上加入 resource facts 與 owner decision 的 typed read model。        | Experience             | repository capability context  | giant optional DTO                 |
| 能力                 | Capability                          | 功能對 resource type 與 canonical action 的註冊描述；本身不授權。                                               | Experience             | product capability             | permission                         |
| 有效能力關係         | Effective Capability                | 對目前 facts 求值後的 `can(Actor, Action, Resource, Context)` 關係；是具體 Authorization Decision 的輸入。      | Resource owner         | capability relation            | UI capability descriptor           |
| 授權決策             | Authorization Decision              | owning resource Context 對 Actor × Action × Resource × Context 回傳的 allow/deny 與 reason。                    | Resource owner         | access decision                | role                               |
| 治理邊界             | Boundary                            | Resource 的 ownership／governance 邊界；不同於 UI Active Scope、Credential Grant 或 Policy Target。             | Account/Resource owner | ownership boundary             | generic scope                      |
| 授與                 | Grant                               | Role、binding、base permission 或 delegation 所提供的候選能力來源。                                             | Grant owner            | authority grant                | policy constraint                  |
| 約束                 | Constraint                          | 依 policy、resource state、credential 或 request condition 縮小候選能力的規則。                                 | Constraint owner       | access constraint              | role grant                         |
| 領域前置條件         | Domain Precondition                 | Authorization allow 後，Resource 目前狀態仍必須滿足的 business invariant。                                      | Resource owner         | operation precondition         | permission check                   |
| 實體參照             | Entity Reference                    | 跨邊界使用的 type + stable ID；只識別 object，不攜帶完整 entity 或通用行為。                                    | Publishing owner       | entity ref                     | shared entity、global aggregate    |
| 關係事實             | Relationship Fact                   | owning Context 發布的 Subject ─ relation → Object 有向事實；不是中央可變 graph row。                            | Relation owner         | relationship tuple             | universal relationship             |
| 資源                 | Resource                            | 具有 identity、lifecycle 並由 owning Context 控制 action 的操作對象。                                           | Resource owner         | target                         | route、page                        |
| 動作                 | Action                              | Resource owner 定義、可供授權與歸因的 canonical operation。                                                     | Resource owner         | requested action               | capability、UI intent              |
| 領域事件             | Domain Event                        | owning Context 對已完成狀態改變發布的不可變事實；不是 command 或通知。                                          | Event producer         | event                          | request、notification              |
| 讀取模型             | Read Model                          | Consumer 依 facts 建立的 purpose-specific query result；不擁有來源 Aggregate。                                  | Query consumer         | query result, view model       | source of truth、product Context   |
| 帳戶                 | Account                             | 擁有資源或治理範圍的 personal、organization、enterprise 容器。                                                  | Account                | personal account, organization | user（泛稱）、login                |
| 企業                 | Enterprise                          | 集中治理一個或多個 Organization 的治理根；不是登入 Principal 或直接 Resource owner。                            | Enterprise Governance  | enterprise account             | organization、repository owner     |
| 企業組織關係         | Enterprise Organization Affiliation | Organization 受特定 Enterprise 治理的關係。                                                                     | Enterprise Governance  | organization affiliation       | organization membership            |
| 企業儲存庫可見性政策 | Repository Visibility Policy        | 對受治理 Organization 的 Public Repository 建立與可見性變更形成上限。                                           | Enterprise Governance  | repository governance policy   | repository permission              |
| 成員邀請             | Membership Invitation               | organization owner 對 Principal 提議建立 Membership 的七日期限流程；可被接受、到期或取消，尚不是有效成員關係。  | Account                | invitation                     | membership、repository grant       |
| 成員邀請狀態         | Membership Invitation Status        | pending invitation 只能轉為 accepted、expired 或 cancelled；終止狀態不可再轉移。                                | Account                | invitation lifecycle           | membership status、delivery status |
| 成員角色             | Membership Role                     | Organization Membership 的封閉角色詞彙 `owner`／`member`；不是 Repository Role。                                | Account                | organization role              | repository role、global role       |
| 成員關係             | Membership                          | Principal 接受有效 Invitation 後與 organization Account 形成的 active／removed 關係事實。                       | Account                | membership                     | repository role、global role       |
| 成員關係狀態         | Membership Status                   | Organization affiliation 的 `active`／`removed` lifecycle；不與 Team membership 或 Principal status 共用。      | Account                | membership lifecycle           | team status、principal status      |
| 團隊                 | Team                                | organization Account 內以 canonical Team Name 聚合 active Membership references；不是 identity 或資源 owner。   | Account                | group                          | account、repository role           |
| 團隊名稱             | Team Name                           | 單一 organization 內唯一的 lowercase canonical name；與 Team stable identity 分離。                             | Account                | team slug                      | display name、account handle       |
| 團隊成員事實         | Team Membership Fact                | Account 對特定 Principal 發布的最小 Team 關係集合；不攜帶 Repository Role。                                     | Account                | team fact                      | repository grant、role             |
| 命名空間             | Namespace                           | Account 所擁有的人類可讀、可路由名稱。                                                                          | Account                | handle                         | repository owner string            |
| 儲存庫               | Repository                          | 由 Personal Account 或 Organization 擁有的產品資源容器。                                                        | Repository             | repo                           | workspace、folder、Git adapter     |
| 擁有者名稱           | Name With Owner                     | `owner login + Repository name` 的可路由名稱；不是 Repository identity。                                        | Repository             | repository address             | RepositoryId                       |
| 儲存庫可見性         | Repository Visibility               | `public`／`private`／`internal` 的 discovery boundary；不是 access role。                                       | Repository             | visibility                     | permission、authentication status  |
| 儲存庫狀態           | Repository State                    | `active`／`archived`／`disabled` 的 Repository lifecycle state。                                                | Repository             | lifecycle state                | visibility、access status          |
| 儲存庫角色           | Repository Role                     | Repository scope 內的 capability bundle；不屬於 Repository Aggregate。                                          | Authorization & Policy | role                           | organization role、global role     |
| 儲存庫存取授與       | Repository Access Grant             | Principal 或 Team 與 Repository 之間的存取關係；不是 Repository 子實體。                                        | Authorization & Policy | grant                          | credential、Membership             |
| 有效存取             | Effective Access                    | 由 identity、relationship、grant、policy 與 Repository facts組合出的特定 action decision。                      | Authorization & Policy | access decision                | login status、single role          |
| 工作項目             | Issue                               | Repository scope 內具有穩定 identity、number 與 open/closed lifecycle 的可追蹤工作。                            | Issues                 | work item                      | Git issue、ticket row              |
| 工作編號             | Issue Number                        | 單一 Repository scope 內單調配置的人類可讀序號；不是 Issue identity。                                           | Issues                 | issue number                   | IssueId、array index               |
| 分類標籤             | Label                               | Repository scope 內名稱唯一的工作分類；不代表 status 或 permission。                                            | Issues                 | classification                 | role、status、free tag             |
| 責任指派             | Assignment                          | Issue 與 eligible Principal 之間可撤銷的工作責任關係。                                                          | Issues                 | assignee relationship          | membership、repository grant       |
| 負責人               | Assignee                            | 持有 active Assignment 的 Principal；authentication 本身不構成 eligibility。                                    | Issues                 | responsible principal          | owner、member                      |
| 討論分類             | Discussion Category                 | Repository scope 內承載 Discussion 並宣告是否可接受 Answer 的分類。                                             | Discussions            | category                       | label、Issue type                  |
| 討論                 | Discussion                          | Repository Category 內的探索性對話 Aggregate；不代表可追蹤工作。                                                | Discussions            | community conversation         | Issue、ticket                      |
| 討論留言             | Discussion Comment                  | Discussion 內由 Principal 發表、可成為 Answer 的內容 Entity。                                                   | Discussions            | comment                        | Issue comment、notification        |
| 採納答案             | Accepted Answer                     | answerable Discussion 對同一 Aggregate 內一個 active Comment 的唯一參照。                                       | Discussions            | marked answer                  | arbitrary comment id、resolution   |

新增 Entity、Value Object、Aggregate、Use Case 或公開 contract 前，必須在本表定義中文名稱、正式 Code Name、定義、Owner Context、允許同義詞、禁止詞與相關概念。

`Issue`、`Issue Number`、`Label`、`Assignment`、`Assignee`、`Discussion Category`、`Discussion Comment`
與 `Accepted Answer` 已由其 Context 擁有並具有 in-memory runtime。其餘 GitHub 非 Code 語意維持
Research／Proposed，完整分類
由 [`../product/capabilities.md`](../product/capabilities.md) 路由；沒有 owner、invariant 與
first use case 前不得建立 runtime contract。
